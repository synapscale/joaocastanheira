# Design System

Este design system documenta todos os componentes da aplicação, organizados seguindo a metodologia Atomic Design.

## Estrutura

O design system está organizado em cinco níveis:

1. **Átomos**: Componentes básicos e indivisíveis
2. **Moléculas**: Grupos de átomos funcionando juntos
3. **Organismos**: Grupos complexos de moléculas
4. **Templates**: Layouts que organizam organismos
5. **Páginas**: Implementações específicas de templates com dados reais

## Princípios de Design

### Consistência
Todos os componentes seguem um conjunto consistente de princípios de design, garantindo uma experiência coesa.

### Acessibilidade
Os componentes são projetados para serem acessíveis a todos os usuários, seguindo as diretrizes WCAG 2.1.

### Responsividade
A interface se adapta a diferentes tamanhos de tela, desde dispositivos móveis até desktops.

### Desempenho
Os componentes são otimizados para carregamento rápido e interações fluidas.

## Como usar este Design System

1. **Explorar componentes**: Navegue pela barra lateral para explorar os componentes disponíveis
2. **Ver exemplos**: Cada componente inclui exemplos de uso e variações
3. **Copiar código**: Use o botão "Show code" para copiar o código do componente
4. **Testar interações**: Interaja com os componentes para ver seu comportamento

## Contribuindo para o Design System

1. **Adicionar componentes**: Crie novos componentes seguindo os padrões existentes
2. **Documentar**: Adicione descrições, exemplos e propriedades para cada componente
3. **Testar**: Garanta que os componentes funcionem em diferentes cenários
4. **Revisar**: Solicite revisão de outros membros da equipe antes de finalizar
